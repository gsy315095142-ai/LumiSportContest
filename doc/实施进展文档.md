## 一、项目改造概览

在现有纯前端 React 项目基础上，新增 Node.js 后端（Express + Socket.IO），实现 PC 端开奖台与手机端竞猜页面的实时通信。数据用 JSON 文件存储，仅局域网内使用。

## 四、技术方案

### 4.1 新增依赖

**前端：**
- `react-router-dom` — URL 路由（区分 PC `/` 和手机 `/mobile`）
- `socket.io-client` — WebSocket 客户端
- `qrcode.react` — 二维码组件

**后端（server/）：**
- `express` — HTTP 服务
- `socket.io` — WebSocket 服务端
- `cors` — 跨域

### 4.2 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/login` | POST | 登录/注册（首次送100币） |
| `/api/user/:name` | GET | 获取用户信息 |
| `/api/checkin` | POST | 每日签到（+100币，每日1次） |
| `/api/watch-ad` | POST | 看广告（+30币，每日5次） |
| `/api/rankings` | GET | 获取段位排行榜 |
| `/api/game` | GET | 获取当前比赛状态、下注情况、参赛状态 |
| `/api/my-bets/:name` | GET | 获取用户当前下注 |
| `/api/contest/status` | GET | 获取参赛状态（红蓝选手、可选项） |
| `/api/contest/confirm` | POST | 确认参赛（body: { name, side }） |
| `/api/contest/cancel` | POST | 撤销参赛（body: { name }） |
| `/api/match-history/:name` | GET | 获取用户历史战绩 |

### 4.3 Socket.IO 事件

| 事件名 | 方向 | 说明 |
|--------|------|------|
| `game:config` | 客户端→服务端 | PC端更新比赛配置（赛事名称、玩法类型，选手由手机端参赛确认） |
| `game:update` | 服务端→全体 | 比赛信息更新（赛事名称、选手、积分、赔率） |
| `game:start` | 服务端→全体 | 比赛开始，锁定下注 |
| `game:settle` | 客户端→服务端 | PC端提交开奖数据 |
| `game:result` | 服务端→全体 | 开奖结果 + 结算信息 |
| `game:next` | 客户端→服务端 | 进入下一局 |
| `game:error` | 服务端→发送者 | 错误信息提示 |
| `bet:place` | 客户端→服务端 | 用户下注 |
| `bet:cancel` | 客户端→服务端 | 用户取消下注 |
| `bet:update` | 服务端→全体 | 下注情况实时同步 |
| `bet:myBets` | 服务端→发送者 | 用户个人下注信息 |
| `rank:update` | 服务端→全体 | 段位排行更新 |
| `contest:confirm` | 客户端→服务端 | 确认参赛 |
| `contest:cancel` | 客户端→服务端 | 撤销参赛 |
| `contest:update` | 服务端→全体 | 参赛状态变更 |
| `contest:error` | 服务端→发送者 | 参赛操作错误提示 |

### 4.4 数据存储

用户数据持久化到 `server/data/users.json`：
```json
{
  "用户名": {
    "name": "用户名",
    "coins": 100,
    "totalWinnings": 0,
    "rating": 1500,
    "lastCheckin": null,
    "adWatchedToday": 0,
    "lastAdDate": null
  }
}
```

比赛状态（含 `redPlayer`、`bluePlayer`）仅在内存中维护（不持久化）。

历史战绩持久化到 `server/data/matchHistory.json`，每局结算后追加记录。

### 4.5 开奖结算规则

- 胜负竞猜：赔率按双方选手积分计算（1.1~2.0），猜中返还 `赔率 × 投注额`
- 平局：退回投注额
- 元素之王：猜中返还 `3 × 投注额`
- 精准总分：猜中返还 `20 × 投注额`
- 精准分差：猜中返还 `15 × 投注额`
- 猜错不返还
- 元素球次数并列时，要求重新录入不同数值

### 4.6 下注规则

- 胜负竞猜：只能猜单边（红方胜或蓝方胜）
- 趣味竞猜：可同时参与所有项目
- 比赛开始前可随意修改下注额度
- 比赛开始后锁定，无法修改
- 下注总额不能超过当前持有竞猜币
- 已确认参赛的用户不能下注，需先撤销参赛
