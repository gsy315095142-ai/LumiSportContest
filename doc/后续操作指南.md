## 一、Unity 端操作（必做）

### 1.1 在 CtrlMatchForm 中绑定选手信息展示

当前 Unity 端已隐藏段位选择，改为展示选手积分与赔率。需在 Prefab 中手动添加展示组件：

| 步骤 | 操作 |
|------|------|
| 1 | 打开 Unity 项目：`D:\gitProject\LumiSports\Server` |
| 2 | 在 Project 中找到 `CtrlMatchForm` 的 Prefab（通常在 `Assets/GameMain/UI/Forms/` 下） |
| 3 | 双击或选中 Prefab 进入编辑 |
| 4 | 在竞猜相关区域（如赛事名称输入框下方、二维码上方）新建一个 **Text** 组件 |
| 5 | 设置 Text 的 RectTransform（位置、大小、字体大小等） |
| 6 | 在 Inspector 中找到 `CtrlMatchForm` 脚本，将新建的 Text 拖到 **Contest Info Text** 字段 |

**效果**：竞猜配置成功后，该 Text 会每 2 秒轮询并显示：
```
红方: 张三 (积分1520 赔率1.35)
蓝方: 李四 (积分1480 赔率1.65)
```

若未绑定，则仅隐藏段位选择，不显示选手信息。

---

### 1.2 检查竞猜联动配置（可选）

确认 `StreamingAssets/ServerConfig.txt` 中已正确配置：

```json
{
  "ContestApiUrl": "http://<竞猜后端IP>:3001",
  "ContestPageUrl": "http://<竞猜前端IP>:5173/mobile",
  "ContestAdminToken": "你的管理员Token"
}
```

- `ContestApiUrl`：竞猜 Node.js 后端地址  
- `ContestPageUrl`：手机端扫码打开的页面地址（用于生成二维码）  
- `ContestAdminToken`：需与竞猜后端 `adminToken` 一致

## 二、联调测试流程

### 2.1 启动顺序

1. **启动竞猜系统**  
   - 在 `LumiSportContest` 目录下双击 `start.bat`  
   - 或分别启动：`cd server && npm run dev`、`npm run dev`

2. **启动 Unity 游戏**  
   - 打开 `D:\gitProject\LumiSports\Server` 项目并运行

### 2.2 建议测试项

| 序号 | 测试项 | 预期结果 |
|------|--------|----------|
| 1 | Unity 点击「开始竞猜」 | 配置成功，二维码显示 |
| 2 | 手机扫码进入竞猜页 | 可看到赛事名称、玩法 |
| 3 | 手机端【我要参赛】选择红/蓝方并确认 | 参赛成功，Unity 端 Text 显示选手名、积分、赔率 |
| 4 | 手机端下注 | 未参赛时可下注；已参赛时提示「要先撤销参赛才能参与竞猜」 |
| 5 | Unity 开始比赛、录入结果、结算 | 结算成功，选手积分更新 |
| 6 | 手机端查看历史战绩 | 可看到过往对阵、比分、积分变化 |

## 三、常见问题

| 问题 | 可能原因 | 处理建议 |
|------|----------|----------|
| 二维码不显示 | ContestPageUrl 或 ContestApiUrl 配置错误 | 检查 ServerConfig.txt，确保 IP 与端口正确 |
| 选手信息不显示 | 未绑定 contestInfoText | 按 1.1 步骤添加并绑定 Text |
| 参赛/下注失败 | 竞猜后端未启动或网络不通 | 确认 start.bat 已启动，手机与 PC 在同一局域网 |
| HTTP 请求失败 | Unity 不允许 HTTP | Player Settings → Other → Allow downloads over HTTP 设为 Always allowed |
