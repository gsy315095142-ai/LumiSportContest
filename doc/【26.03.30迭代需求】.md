# 【26.03.30迭代需求】

> 手机端弹幕输入体验修正，实施日期：2026-03-30

---

## 一、弹幕输入不支持换行

- **需求**：手机端输入弹幕时不支持换行，保持单行内容。
- **实现**：
  - 输入时拦截 `Enter`，阻止在输入框内插入换行；
  - 输入与提交前均进行单行规范化（将 `\r` / `\n` 替换为空格），避免粘贴多行文本绕过限制。
- **结果**：发送到后端的弹幕文本统一为单行。

---

## 二、输入框提示文案调整

- **需求**：弹幕输入框提示文案仅显示「请输入弹幕内容」。
- **实现**：更新弹幕输入框 `placeholder` 为固定文案：
  - `请输入弹幕内容`
- **结果**：移除“系统表情”相关描述，文案更简洁。

---

## 三、键盘弹起时底部 3 按钮不随之上移

- **问题**：在部分手机 WebView/浏览器中，弹出键盘后底部导航（竞猜 / 发弹幕 / 参赛）会被整体顶起。
- **实现**：优化底栏位移计算逻辑，优先基于稳定布局高度与 `visualViewport` 差值计算补偿位移，兼容 `innerHeight` 同步变化场景。
- **结果**：键盘弹起时底部 3 按钮保持贴底，不再被抬升。

---

## 四、弹幕敏感词能力升级（外置词库 + 热重载）

- **目标**：在保持“命中即拒绝发送”规则不变的前提下，提升敏感词覆盖率与维护效率。
- **实现**：
  - 将敏感词从服务端硬编码迁移到外置文件：`server/data/danmakuBlockedWords.txt`；
  - 服务端改为启动加载词库，支持注释行与去重清洗；
  - 新增热重载接口：`POST /api/danmaku/reload-words`，修改词库后无需重启服务；
  - 批量引入开源词库并清洗合并，当前词条规模约 **2000+**。
- **规则**：
  - 命中任一敏感词直接拦截，返回 `内容包含不当词语`；
  - 频控与长度规则保持不变（5 秒 1 条、长度 1~40）。
- **结果**：敏感词覆盖显著提升，词库可持续运营维护。

## 五、选手积分与胜负竞猜规则（补充同步）

### 5.1 按玩法独立积分

- 选手积分按玩法独立维护：
  - `hockey`（疾速冰球）
  - `boxing`（烈焰拳王）
  - `fencing`（雷霆击剑）
- 历史积分已统一重置为初始值（每玩法 `100`），竞猜币相关数据保留。

### 5.2 当前结算公式（`p = 20`）

- 我方获胜：
  - `新积分 = 旧积分 + p * 对手积分 / (对手积分 + 自己积分)`
- 我方失败：
  - `新积分 = 旧积分 - p * 对手积分 / (对手积分 + 自己积分)`
- 积分下限：结算后最小值为 `1`。

### 5.3 胜负竞猜禁用规则

- 任意一方预期胜率 `>= 81%` 时，禁用胜负竞猜（趣味竞猜不受影响）。
- 机器人对局同样禁用胜负竞猜。
- 机器人定义：**选手名不在用户数据中**。
- 命中“胜负悬殊”场景时，双方赔率不显示具体值，统一显示 `[胜负悬殊]`。

### 5.4 赔率范围

- 胜负赔率保留手续费公式后，最终强制钳制在 `[1.1, 2.0]` 区间。

### 5.5 击剑玩法竞猜范围

- 击剑玩法开放：
  - 胜负竞猜
  - 精准总分
  - 精准分差
- 击剑玩法暂不开放：
  - 元素之王
  - 躺平之王

### 5.6 Unity 端改造方案（供 `D:\gitProject\LumiSports\Server` 开发同学）

- `ContestSyncManager.cs`
  - `TriggerConfigureContest()` 增加 `Fencing -> fencing` 的 `gameType` 映射。
  - `SettleContest()` 增加 `EGame.Fencing` 分支，击剑结算仅上报 `redScore/blueScore`。
  - `FormatContestGameInfoDisplay()` 支持 `[胜负悬殊]` 文案显示。
- `CtrlMatchForm.cs`
  - 确认击剑模式下竞猜配置流程可走通（开始竞猜、锁盘、结算、重置）。
- `Match1Form.cs`（若维护玩法说明文案）
  - 击剑玩法竞猜提示改为“胜负 + 精准总分 + 精准分差”。
- 联调检查项
  - 击剑场次 `/api/game` 返回 `gameType=fencing`。
  - 手机端击剑场次不出现元素之王、躺平之王入口。
  - 击剑结算后仅更新 `ratings.fencing`，不影响其他玩法积分。
